# System Prompt Tool Guidance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add behavioral guidance to the system prompt for seven underdocumented tools — `list_directory`, `cron_create/list/delete`, `lsp`, `web_fetch`, `task_get`, and `task_stop` — and update the task management section to cover the full task lifecycle.

**Architecture:** All changes land in one file: `packages/core/src/core/prompts.ts`. The static system prompt's "Tool Usage" section gets new bullet-point entries for each gap. The task management section adds `task_get` and `task_stop` to its existing guidance. No new files, no new abstractions — the existing pattern of inline guidance strings is followed throughout. Snapshot tests must be updated to reflect the new content.

**Tech Stack:** TypeScript, Vitest (test framework), `npm run test` to run unit tests, `npm run typecheck` for type checking.

**Key context before starting:**

- `packages/core/src/core/prompts.ts` exports `getCoreSystemPrompt()` which returns `{ staticPrefix, dynamicSuffix, full }`.
- The `staticPrefix` is a template-literal string built inside that function. All guidance lives there.
- `ToolNames` constants (imported at the top of `prompts.ts`) must be used instead of bare string literals — e.g., `${ToolNames.LS}` not `'list_directory'`.
- Tests use Vitest. Run with: `npm run test -- --testPathPattern=prompts` (from repo root).
- Snapshots live in `packages/core/src/core/__snapshots__/prompts.test.ts.snap`. When you change prompt content, delete this file and let Vitest regenerate it (see Task 3 Step 2).
