DATA AGENT MATURITY MODEL

数据飞轮 × 自动驾驶 L1-L5 映射

用自动驾驶分级表达 Data Agent 从“知识辅助”到“目标自治”的演进路径，并明确每一级的数据飞轮建设要求。

L1 辅助 L2 诊断 L3 闭环 L4 执行 L5 自治
自动驾驶类比 Data Agent 阶段 核心特征 人的角色 数据飞轮要求
L1辅助驾驶 查询解释型 Agent Agent 提供解释、检索、建议，不主动完成任务。 人主导 记录用户意图、问题类型、答案采纳 / 未采纳；沉淀知识缺失和高频问法。
L2部分自动驾驶 诊断辅助型 Agent Agent 可调用只读工具、给出诊断和证据链。 人确认 记录工具调用链、上下文来源、诊断结果、采纳 / 修改 / 拒绝；归因工具失败、上下文缺失、意图识别错误。
L3条件自动驾驶 场景闭环型 Agent 在特定任务类型内可完成完整诊断 / 方案生成。 人兜底接管 建立标准 Telemetry、失败归因、Eval 回流；每类任务沉淀 Task Card / Tool Card / Rule Card。
L4高度自动驾驶 受控自动执行型 Agent 在白名单场景中可自动执行低风险动作，并有回滚 / 审批 / 监控。 人处理例外 自动采集执行结果、验证结果、回滚 / 审批记录、线上工具成功率与超时率；持续优化权限策略和自动化边界。
L5完全自动驾驶 泛化自治型 Data Agent 跨场景自主理解、执行、验证、优化，持续自我进化。 人只设目标和边界 实现跨场景自动归因、自动 Eval 生成、知识健康度扫描、策略自优化；从业务结果和线上运行数据中持续学习。
核心判断：Phase 1-4 是数据来源成熟度，L1-L5 是任务自治成熟度；两者相关但不是一一对应。
