## Summary

- Add `/history collapse-on-resume` to persist collapse preference for future resumed sessions
- Add `/history expand-on-resume` to persist expand preference for future resumed sessions
- Add `/history expand-now` to expand currently collapsed history in the active session
- Implement `ui.history.collapseOnResume` setting with schema and documentation
- Full test coverage for new commands and resume behavior

## Why

When resuming a long conversation (50+ messages), the terminal scrolls through all historical messages which is disruptive. This PR adds a persistent preference to collapse history by default on resume, with explicit commands to control the behavior.

## How

- Added `ui.history.collapseOnResume` boolean setting (default: false)
- Three new subcommands under `/history`:
  - `collapse-on-resume`: Sets preference to collapse on future resumes
  - `expand-on-resume`: Sets preference to expand on future resumes
  - `expand-now`: Expands currently collapsed history in the active session
- When `collapseOnResume` is true, resumed items are marked with `display.suppressOnRestore = true`
- Suppressed items are filtered from UI rendering but kept in canonical history for `/rewind`
- A summary item shows the count of hidden messages
