# Shell Kill Tool 手动测试用例

## 测试目标

验证 `shell_kill` 工具能够正确终止通过 `run_shell_command` 启动的后台进程。

---

## 测试环境

- **项目**: qwen-code
- **分支**: fix/issue-2555-pty-ebadf
- **测试模式**: 交互模式 (QWEN_SANDBOX=false)

---

## 测试用例

### 测试 1: 基本终止功能 (SIGTERM)

**步骤：**

1. 启动 qwen-code 交互模式
2. 启动一个后台进程（如 sleep 或 node 服务器）
3. 获取返回的 PID
4. 使用 `shell_kill` 终止该进程
5. 验证进程已被终止

**测试命令：**

```bash
cd ~/code/qwen-code
npm run start
```

**对话示例：**

```
用户：启动一个后台进程，运行 sleep 1000
AI: 后台命令已启动。PID: 12345

用户：终止进程 12345
AI: 已发送 SIGTERM 信号到 PID 12345。进程应该很快终止。
```

**预期结果：**

- ✅ 返回成功消息，包含 PID 和信号类型
- ✅ 进程实际被终止（可通过 `ps` 验证）

---

### 测试 2: 强制终止 (SIGKILL)

**步骤：**

1. 启动一个不响应 SIGTERM 的进程
2. 使用 `shell_kill` 发送 SIGKILL 信号

**测试命令：**

```
用户：终止进程 12345，使用强制终止
```

**预期结果：**

- ✅ 返回 "已发送 SIGKILL 信号到 PID 12345"

---

### 测试 3: 终止不存在的进程

**步骤：**

1. 尝试终止一个不存在的 PID

**测试命令：**

```
用户：终止进程 99999
```

**预期结果：**

- ✅ 返回错误："未找到 PID 为 99999 的 shell 会话"

---

### 测试 4: 终止已退出的进程

**步骤：**

1. 启动一个快速退出的后台进程
2. 等待进程自然退出
3. 尝试终止该进程

**预期结果：**

- ✅ 返回错误："PID 为 xxx 的 shell 会话已退出，退出码：0"

---

### 测试 5: 完整流程测试

**场景：** 启动一个 HTTP 服务器，然后终止它

**测试命令：**

```bash
cd ~/code/qwen-code
npm run start
```

**对话流程：**

```
用户：在当前目录启动一个 HTTP 服务器，使用 npx http-server -p 8080，后台运行

AI: 后台命令已启动。PID: 54321

用户：检查端口 8080 是否可访问

AI: [验证服务器运行中]

AI: 已发送 SIGTERM 信号到 PID 54321

用户：检查端口 8080

AI: [验证服务器已停止]
```

**预期结果：**

- ✅ 服务器成功启动
- ✅ 服务器成功终止
- ✅ 端口被释放

## 验证命令

### 验证进程是否存在

```bash

```

### 验证端口是否被占用

```bash

# 或
netstat -tlnp | grep 8080
```

## 常见问题排查

### 问题 1: 工具未注册

**症状：** AI 无法识别 `shell_kill` 命令
**解决：** 确认已运行 `npm run build` 并使用最新版本

### 问题 2: PID 不匹配

**症状：** 找不到对应的进程
**解决：** 确认 PID 是通过 `run_shell_command` 启动的后台进程

### 问题 3: 权限问题

**症状：** 无法终止进程
**解决：** 确认进程属于当前用户，或有足够的权限
