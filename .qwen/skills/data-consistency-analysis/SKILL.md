## 任务概述

当发现两个数据表（通常为下游表）数据不一致时，需要进行系统性分析，通过血缘追踪、字段对比、上游依赖分析等手段定位差异的根本原因。

## 分析流程

### 1. 表信息识别

- 使用 `identify.py` 获取表的详细档案（字段、分区、数据量、Owner 等）
- 确认表的所属工作空间和数据库

### 2. 血缘关系分析

- 使用 `query_lineage.py` 分析表的上下游关系
- 获取上游表列表及节点 ID
- 对比不同表的上游依赖差异

### 3. 字段对比

- 使用 `query_columns.py` 获取两个表的完整字段列表
- 对比字段数量、字段名、类型和注释差异

### 4. 上游依赖分析

- 深入分析上游表的血缘关系
- 确定是否存在中间表（如 ADS 层）
- 对比不同路径的上游数据源

### 5. 问题定位

- 结合血缘关系和字段差异，判断可能的数据差异原因
- 重点关注：
  - 不同的数据源引入（如额外的 ODS 表）
  - 不同的 JOIN 逻辑和过滤条件
  - 不同的业务维度映射
  - 数据量上的显著差异

## 适用场景

- 发现下游表数据量、字段或内容不一致
- 需要排查数据质量问题
- 跨层级表间数据差异分析（如 ODS → DWD → ADS）

## 注意事项

- 由于权限限制，有时无法直接查看节点 SQL，此时需要依赖血缘分析和元数据
- 血缘分析可能存在截断，需关注是否达到最大搜索深度
- 建议结合实际业务逻辑和数据模型理解差异原因
