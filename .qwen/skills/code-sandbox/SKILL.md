## Когда использовать

- Перед push кода
- Для проверки что тесты проходят
- Для тестирования исправлений багов
- Для проверки производительности

## Что делает

1. Запускает код в изолированном процессе
2. Использует пониженный приоритет (start /belownormal)
3. Ограничивает время выполнения
4. Собирает метрики (время, память, ошибки)

## Как использовать

```
Примени skill: code-sandbox для запуска tests
```

## Безопасность

- Нет доступа к сети (если не разрешено)
- Ограниченный доступ к файловой системе
- Таймаут 5 минут
- Пониженный приоритет CPU

## Команды Windows для запуска с низким приоритетом

```powershell
Start-Process -FilePath "npm" -ArgumentList "test" -WindowStyle Hidden -PassThru | ForEach-Object { $_.PriorityClass = "BelowNormal"; $_.WaitForExit() }
```
