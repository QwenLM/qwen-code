## When you sync to a newer upstream ref

`scripts/sync-from-upstream.sh` diffs `.vendored-from`→`<new-ref>`. If `<new-ref>`
is **at or past the commit where one of these PRs merged**, that PR's changes are
already in the upstream delta, so:

1. The hunks will collide with the cherry-picked commit (expect `.rej` on the
   files above), and
2. Once reconciled, **remove the row from the active table** — it's no longer a
   local patch, it's part of the base.

If `<new-ref>` is still *before* a PR merges, that row stays; the cherry-pick
keeps carrying the fix.
