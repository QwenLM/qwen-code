## 🔍 集成测试注意事项

### 关键限制

1. **会话隔离**: 每次运行 CLI 都是独立会话，后台进程不共享
2. **测试方式**: 必须在同一个交互会话中测试启动和终止
3. **PID 范围**: 只能终止通过 `run_shell_command` 启动的进程

### 正确的测试方法

```bash
# ❌ 错误：两次独立的 CLI 调用
node dist/cli.js -p "启动后台进程 sleep 300"
node dist/cli.js -p "终止进程 XXXXX"  # 找不到会话

# ✅ 正确：同一个交互会话
node dist/cli.js --yolo --sandbox=false
# 然后交互式输入命令
```
